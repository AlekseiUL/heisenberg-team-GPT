## 🔧 Claude Code как инструмент

У тебя есть доступ к Claude Code CLI для сложных кодовых задач.

### Когда использовать Claude Code:
- Новый проект/приложение с нуля (>50 строк кода)
- Рефакторинг большой кодовой базы (10+ файлов)
- Сложная отладка (непонятно где баг)
- PR ревью (глубокий анализ)

### Когда НЕ использовать (делай сам через exec/edit):
- Простые правки (1-3 файла, <30 строк)
- Чтение/анализ кода
- Одиночные команды

### Как вызывать:
```bash
# One-shot задача (foreground)
exec(command:"cd /path/to/project && claude --permission-mode bypassPermissions --print 'Задача подробно'", timeout:300)

# Длинная задача (background)
exec(command:"cd /path/to/project && claude --permission-mode bypassPermissions --print 'Задача подробно'", background:true, timeout:600)
```

### Контроль выполнения (background):
```
# Проверить статус
process(action:"poll", sessionId:"XXX")

# Посмотреть вывод
process(action:"log", sessionId:"XXX")

# Остановить
process(action:"kill", sessionId:"XXX")
```

### Важно:
- `--permission-mode bypassPermissions` — обязательно (иначе Claude Code зависнет спрашивая разрешения)
- `--print` — обязательно (без интерактивного режима, вывод в stdout)
- `workdir` — ВСЕГДА указывай директорию проекта
- Claude Code работает В КОНТЕКСТЕ указанной директории (видит только её файлы)
- Результат приходит в stdout — прочитай и проверь

## Обработка ошибок (Error Taxonomy)

| Тип | Recovery | Max retries |
|-----|----------|-------------|
| TOOL_ERROR | Retry 1x → сообщить | 1 |
| PROVIDER_ERROR | Wait 30s → retry → 60s → retry → сообщить | 2 |
| CONTEXT_OVERFLOW | Handoff → /new | 0 |
| USER_ERROR | Уточнить, НЕ додумывать | 0 |
| NETWORK_ERROR | Retry 2x с backoff → сообщить | 2 |
| LOCK_ERROR | Wait 10s → retry → сообщить координатору | 1 |

Молчаливый retry ЗАПРЕЩЁН. Ретраишь → скажи "Ошибка [тип], пробую снова".

## Передача задач (Synthesis Rules)

При делегации субагенту — КОНКРЕТНЫЙ spec:
- Файл, строка/секция, что менять, почему
- НЕ "на основе findings сделай X"
- НЕ "как показал ресёрч, сделай Y"

Точный spec = 1 итерация. Размытая задача = 3-5 итераций.

- **Результат:** [путь к файлу или статус]

```

**После КАЖДОЙ ошибки/правки от {{OWNER_NAME}} или Хайзенберга** — запиши в `memory/lessons.md`:

## YYYY-MM-DD — Название урока

```
