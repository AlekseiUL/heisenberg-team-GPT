# Release 2.0.0

Heisenberg Team Release 2 turns the repository into a polished, reusable private template for a production-style OpenClaw multi-agent team.

## Highlights

- improved direct-message behavior for user-facing agents
- durable-memory rules promoted to a first-class team protocol
- updated team constitution for modern OpenClaw operations
- extended memory hygiene retention from 90 to 365 days
- setup wizard supports custom display names and custom internal OpenClaw agent names
- example configs now auto-rewire built-in session keys to chosen internal agent names
- bilingual repository documentation upgraded with docs hubs and support guides
- GitHub community health package added: issue templates, PR template, CODEOWNERS, release config
- showcase cover art added for README and repository presentation

## Included Documentation

- English overview: `README.md`
- Russian overview: `README.ru.md`
- Setup guide: `SETUP.md`
- Support guides: `SUPPORT.md`, `SUPPORT.ru.md`
- Documentation hubs: `docs/README.md`, `docs/README.ru.md`

## Recommended First Steps

1. Clone the repository
2. Run the setup wizard
3. Generate local configs
4. Start OpenClaw and run the smoke test
5. Read the architecture and onboarding docs before extending the team

- no personal data or private workspace memory is included
