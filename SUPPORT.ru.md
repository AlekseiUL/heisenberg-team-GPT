# Поддержка

Нужна помощь по Heisenberg Team? Здесь краткий и нормальный маршрут без лишнего шума.

## Что проверить до создания issue

1. Прочитайте [README.ru.md](README.ru.md)
2. Пройдите [SETUP.md](SETUP.md)
3. Откройте [docs/README.ru.md](docs/README.ru.md)
4. Проверьте [docs/faq.md](docs/faq.md)
5. Запустите локальный smoke test:

```bash
bash scripts/smoke-test.sh
```

## Куда писать в зависимости от проблемы

### Баг
Создайте GitHub Issue и приложите:
- что ожидали увидеть
- что произошло по факту
- точную команду или путь к файлу
- версию OpenClaw
- ОС и shell
- обезличенные логи или скриншоты

### Пробел в документации
Создайте GitHub Issue и укажите:
- какую страницу вы читали
- какой шаг был непонятен или пропущен
- на какой команде или экране вы застряли

### Проблема установки
Добавьте:
- вывод `openclaw status`
- вывод `bash scripts/smoke-test.sh`
- использовали мастер или ручную установку
- это свежая установка или attach-existing сценарий

### Security-проблема
Не публикуйте её открыто. Используйте приватный процесс из [SECURITY.md](SECURITY.md).

## Что обезличить перед публикацией

Удалите или замените:
- API keys
- bot tokens
- личные имена, ID, email
- приватные пути, если они раскрывают личность
- внутренние chat links или invite links

Используйте плейсхолдеры `{{TOKEN}}`, `{{OWNER_NAME}}` и `<workspace-path>`.

## Что входит в scope проекта

Поддержка распространяется на:
- мультиагентный шаблон для OpenClaw
- роли команды, скиллы и правила координации
- скрипты установки, онбординга и обслуживания

Не входит в scope:
- сбои сторонних API
- приватные форки без воспроизводимого кейса
- проблемы биллинга у провайдеров

## Рекомендуемый порядок диагностики

1. `openclaw status`
2. `bash scripts/smoke-test.sh`
3. `docs/faq.md`
4. `docs/linux-setup.md`, если у вас Linux или WSL
5. GitHub Issue с обезличенными деталями

## По ожиданиям ответа

Поддержка ведётся по best-effort модели. Для быстрой реакции дайте минимально воспроизводимый кейс и чистые логи.
